Placeholder page template

Genesis

Two clients with WordPress websites needed to use Placeholder pages so that Breadcrumbs, which rely on the Page Hierarchy, would match the Menu structure.

For the first client, I set up Menu Items with a custom URL of "#". In that case, clicking on the Menu Item causes the user to stay on the current page.

For the second client, I found a snippet of code that could be put in a template's "page-templates" directory that would direct the user to the first child in the Page Hierarchy. However, it did not take into account that the page might have no children in the Page Hierarchy, even though there were child Menu Items in the Menu Hierarchy. This can happen when child Menu Items are Category Pages or Custom Post Types.

Development

The first round of development was to create the code for a Placeholder page template that finds and redirects to the first child Menu Item. (Note that this works for multi-level redirection.) This code has been installed on the second client's site, and works quite well. The code has been placed in a Github repository for safekeeping.

The second round of development will be to create a WordPress plugin installation procedure. The plugin will have an administrative interface that allows the site administrator to configure the page template on a per-theme basis. The interface will obscure those themes which do not have a "page-template" directory and provide options for the site administrator in the event of a name conflict.
